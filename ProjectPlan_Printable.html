KẾ HOẠCH PHÁT TRIỂN DỰ ÁN

Hệ thống Quản lý Xe buýt Trường học (School Bus Management System)

1. Công nghệ & Kiến trúc Hệ thống

  • Nền tảng Controller: Java Servlets
  • Views: JSP (JavaServer Pages), JSTL, Bootstrap 5
  • Tương tác Database: JDBC (DAO Pattern)
  • Cơ sở dữ liệu: SQL Server (SchoolBusDB)
  • Web Server: Apache Tomcat

2. Phân chia công việc (Nhóm 5 thành viên)

Thành viên 1: Trưởng nhóm & Hạ tầng (Integrator)
  • Thiết lập cấu trúc Project, cấu hình DBContext.
  • Xây dựng Module Authentication (Login, Logout, Cookie, Phân quyền).
  • Thiết kế Layout chung (Header, Sidebar, Footer).
  • DAO: UserDAO (phần xác thực).
Thành viên 2: Admin - Quản lý Nhân sự
  • CRUD Users: Quản lý tài khoản Admin, Driver, Monitor, Parent.
  • CRUD Students: Quản lý thông tin học sinh và liên kết với Phụ huynh.
  • DAO: UserDAO (phần CRUD), StudentDAO.
Thành viên 3: Admin - Quản lý Vận hành
  • CRUD Routes & Stops (Tuyến đường & Điểm dừng).
  • CRUD Vehicles (Xe buýt).
  • Logic Phân công & Giờ chạy (Trips): Tự động sinh lịch trình.
  • DAO: RouteDAO, VehicleDAO, AssignmentDAO.
Thành viên 4: Module Giám sát (Monitor)
  • Monitor Dashboard: Hiển thị danh sách chuyến đi trong ngày.
  • Attendance: Điểm danh học sinh theo thời gian thực.
  • DAO: TripDAO, AttendanceDAO.
Thành viên 5: Module Phụ huynh & Tài xế
  • Parent Portal: Theo dõi xe của con, báo nghỉ (Absence).
  • Driver UI: Cập nhật trạng thái xe (Bắt đầu/Hoàn thành).
  • Kiểm thử (QA): Kiểm tra lỗi tổng thể.
  • DAO: AbsenceDAO.

3. Lộ trình Triển khai

Giai đoạn Nội dung trọng tâm
GĐ 1 Database, Login Flow, Common Layout
GĐ 2 Hệ thống Admin (Quản lý dữ liệu lõi)
GĐ 3 Nghiệp vụ Monitor, Điểm danh & Tài xế
GĐ 4 Cổng thông tin Phụ huynh & Testing hoàn thiện

4. Checklist quan trọng phòng lỗi

  • Mọi Servlet phải kiểm tra Session (Redirect nếu chưa login).
  • Sử dụng PreparedStatement để chống SQL Injection.
  • Đóng Connection/ResultSet trong khối finally tại lớp DAO.
  • Kiểm tra đường dẫn nội bộ (/WEB-INF/...) khi Dispatcher.
